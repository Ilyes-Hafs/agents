01 — Overview
System Architecture
How everything fits together
Router
qwen2.5:3b
The dispatcher. Reads every incoming message and classifies it into a category with a single word. Lightweight and fast — designed to load instantly and stay out of the way.
Always loads first
Prompt Engineer
llama3.2:3b
Automatically rewrites short or vague inputs into clear, detailed prompts before routing. Only activates on messages under 10 words — longer inputs pass through unchanged.
Auto on short inputs
Coding
qwen2.5-coder:7b
A code-specialized model trained on a large corpus of programming languages. Writes, debugs, explains, and refactors code across Python, JS, C, Rust, shell scripts, and more.
4.7 GB
Math & Reasoning
deepseek-r1:7b
A distilled reasoning model that thinks step-by-step before answering. Excels at math problems, logic puzzles, and multi-step analysis. Shows its reasoning chain explicitly.
4.7 GB
General + Search
llama3.2:3b
A fast, capable general-purpose assistant. Handles writing, translation, summarization, synthesizing web search results, and powering the prompt improver.
2.0 GB
ℹ
The three-stage pipeline
Every message goes through up to three stages: (1) the Prompt Improver rewrites short inputs into better prompts, (2) the Router classifies the improved message into one word: CODE, MATH, SEARCH, TTS, FILE, AIDER, or GENERAL, and (3) the appropriate specialist model loads and handles the request. Only one model occupies VRAM at a time.

03 — AI Agents
main.py — The Brain
Terminal-based multi-agent system with automatic prompt improvement

The main agent system is the primary terminal interface for all AI tasks. It uses a three-stage pipeline: your input is first improved by a prompt engineer, then classified by the router, then handled by the appropriate specialist model. Built with CrewAI and Ollama.

The pipeline

Stage 1
🪄 Prompt Improver
If your message is under 10 words, llama3.2:3b automatically rewrites it into a clearer, more detailed prompt before anything else runs. You see the improved version printed as [PROMPT] →.
Auto on short inputs
Stage 2
🔀 Router
qwen2.5:3b reads the (possibly improved) message and outputs one word to select the right specialist: CODE, MATH, SEARCH, TTS, FILE, AIDER, or GENERAL.
Always runs
Stage 3
🎯 Specialist
The appropriate model handles the request — coder for code, reasoning model for math, search-augmented general for current info, or one of the tool agents for file/TTS/aider tasks.
Specialist loads on demand

How to use it

1
Activate the virtual environment and start
Run source ~/.venvs/agents/bin/activate.fish then python ~/agents/main.py. The agents shell alias handles both steps automatically.
2
Ask anything naturally
No need to craft perfect prompts or specify which model to use. Short vague inputs get improved automatically. Detailed inputs pass through unchanged.
3
Watch the pipeline labels
You'll see [PROMPT] → if your input was improved, then [Router → CATEGORY] showing which specialist handled it.
4
Type exit to quit
Or press Ctrl+C. The session ends and the virtual environment deactivates.

Example session

terminal — example session with prompt improver 
# Start the system
python ~/agents/main.py

🤖 Local AI ready. Type your question (or 'exit'):

# Short input — prompter activates automatically
You: explain recursion
[PROMPT] → Provide a clear explanation of recursion in programming,
           including how it works, a simple Python example, and
           common pitfalls like missing base cases.
[Router → CODE]

# Detailed input — prompter skips, passes through unchanged
You: write a Python function that reads a CSV file and returns the mean of a specific column
[Router → CODE]

# Search with short input — prompter improves then searches
You: latest python version
[PROMPT] → What is the latest stable release version of Python as of today?
[Router → SEARCH]
[Searching...]

You: exit
ℹ
Prompt improver threshold
The prompter only activates on messages under 10 words. This avoids slowing down requests where you already provided enough detail. The threshold can be adjusted in main.py by changing the word count check in maybe_improve_prompt().
⚠
First response is slower
On first run, the router model takes 2–4 seconds to load into VRAM. If the prompter also triggers, expect an additional 1–2 seconds. Subsequent questions in the same session are faster.

05 — SearXNG
Private Search Engine
Anonymous web search + AI integration

SearXNG is a self-hosted, privacy-respecting meta search engine. Rather than querying Google directly (which tracks your searches), SearXNG acts as a proxy — it fetches results from Google, Bing, DuckDuckGo, and dozens of other sources simultaneously, then returns them to you with no tracking, no ads, and no profiling. It also serves as the data source for the SEARCH agent in main.py and the web search feature in Open WebUI. → SearXNG docs

Three ways to use it

Direct browser
Manual Search
Navigate to localhost:8888 and use it like any search engine. Results come from multiple engines simultaneously with no tracking whatsoever.
localhost:8888
Via main.py
AI-Synthesized Search
Ask time-sensitive questions in main.py. The SEARCH route queries SearXNG, retrieves the top results, and feeds them to llama3.2:3b which synthesizes a coherent answer.
Auto-routed
Via Open WebUI
Browser-Based AI Search
Toggle the 🌐 web search icon in WebUI before sending a message. WebUI queries SearXNG and augments the model's response with live results.
Toggle per message
ℹ
When to use search vs when not to
Use search for anything time-sensitive: current news, software versions, prices, recent events, live sports scores, weather. Do not use search for timeless knowledge: programming concepts, math, definitions, history — the local models already know these and search would only slow things down.
✓
Auto-starts via Docker
SearXNG runs in a Docker container configured with restart: always. It starts automatically when Docker starts on boot and requires no manual intervention.

07 — ComfyUI
Image Generation
Stable Diffusion XL Turbo on your 4060

ComfyUI is a node-based image generation interface for Stable Diffusion. Instead of a simple form, it exposes the entire image generation pipeline as a visual graph — boxes (nodes) connected by wires (edges). Each node does one job: load a model, encode your text, sample noise, decode pixels. Your loaded model is SDXL-Turbo — optimized for speed, generates 1024×1024 images in a few seconds on your RTX 4060. → ComfyUI GitHub

How nodes work

Think of it like a factory assembly line. Raw materials (your text prompt, a model) go in one end, a finished image comes out the other. Each station (node) transforms the data in one specific way and passes it to the next via colored connection wires.

Purple wires
Model data
Carries the loaded checkpoint model between nodes. Starts at Load Checkpoint and flows into KSampler.
MODEL
Yellow wires
Conditioning
Carries your encoded text prompt. Flows from CLIP Text Encode nodes into KSampler as positive/negative guidance.
CONDITIONING
Blue wires
Latent image
Carries the compressed latent representation of the image. Flows from Empty Latent Image → KSampler → VAE Decode.
LATENT
Green wires
Pixel image
Carries the final decoded pixel image. Flows from VAE Decode → Save Image. This is what you see.
IMAGE

The standard SDXL node pipeline

Your sdxl_workflow.json contains exactly these nodes wired together:

#NodeWhat it doesKey settings
1 Load Checkpoint Loads the SDXL-Turbo model file from disk into VRAM ckpt_name: your model file
2 CLIP Text Encode (Positive) Converts your text prompt into numbers the model understands text: your description here
3 CLIP Text Encode (Negative) Tells the model what to avoid in the image text: blurry, ugly, deformed...
4 Empty Latent Image Creates a blank canvas of the right size in latent space width: 1024, height: 1024, batch: 1
5 KSampler The core — iteratively denoises the latent using your prompt as guidance steps: 1-4, cfg: 1.0, sampler: euler_a
6 VAE Decode Converts the latent image back into actual pixels Automatic — no settings to change
7 Save Image Saves the final PNG to ~/ComfyUI/output/ and shows it in the UI filename_prefix: your prefix

KSampler settings explained

The KSampler is the most important node — it controls how the image is generated. → Sampler guide

SettingWhat it doesFor SDXL-Turbo
steps How many denoising iterations to run. More = slower but more refined. 1–4 (turbo is designed for very low steps)
cfg (CFG Scale) How strictly to follow your prompt. Higher = more literal, less creative. 1.0 (SDXL-Turbo uses distillation, high CFG breaks it)
sampler_name The algorithm used for denoising. euler_a or dpm_adaptive
scheduler Controls the noise schedule across steps. karras or normal
denoise How much of the image to regenerate. 1.0 = full generation from noise. 1.0 for new images, lower for img2img
seed Random seed. Same seed + same prompt = same image. Use -1 for random, or fix a seed to reproduce results

How to generate an image

1
Start ComfyUI
Type comfyui in terminal. It opens localhost:8188 in your browser automatically. Close AI agents first to free VRAM.
2
Load your workflow
Click Load (or drag and drop) ~/agents/sdxl_workflow.json onto the canvas. The node graph appears.
3
Edit the positive prompt
Click the CLIP Text Encode (Positive) node and type your description in the text box. Be specific and descriptive.
4
Edit the negative prompt
Click the CLIP Text Encode (Negative) node. Add things to avoid: blurry, low quality, ugly, distorted, deformed, watermark.
5
Click Queue Prompt
Press Queue Prompt (or Ctrl+Enter). Watch the progress bar on the KSampler node. Image appears on Save Image node when done.
6
Find your image
Right-click the Save Image node → Open in folder, or find it directly at ~/ComfyUI/output/.

Prompt writing tips for SDXL-Turbo

→ SDXL prompt guide

prompt examples 
# Good positive prompt structure:
[subject], [setting], [lighting], [style], [quality tags]

# Example — photorealistic portrait:
a young woman with auburn hair, sitting in a cafe, golden hour lighting,
bokeh background, photorealistic, 8k, sharp focus, cinematic

# Example — landscape:
misty mountain valley at dawn, pine forest, purple sky, volumetric fog,
epic scenery, landscape photography, ultra detailed

# Example — artwork style:
a knight in enchanted forest, oil painting style, dramatic lighting,
fantasy art, intricate details, vibrant colors

# Standard negative prompt (always use this):
blurry, low quality, ugly, distorted, deformed, watermark, text,
extra limbs, bad anatomy, jpeg artifacts, noise, overexposed
ℹ
SDXL-Turbo quirks
SDXL-Turbo is a distilled model — it was trained to generate good images in 1–4 steps instead of the usual 20–50. Because of this: keep CFG at 1.0 (not the usual 7), use euler_a sampler, and don't go above 4 steps or quality actually decreases.
⚠
Manual start only — VRAM conflict
ComfyUI uses ~6GB VRAM when running. Your AI models need 2–4.5GB each. Running both simultaneously will cause out-of-memory errors on your 8GB card. Always close your agents session before starting ComfyUI.
ℹ
Best resolution for your GPU
Stick to 1024×1024. Maximum safe resolution is ~1280×1280. Going higher risks OOM errors on 8GB VRAM. Change resolution in the Empty Latent Image node.

12 — Limits
Context & Processing Limits
How much each tool can handle at once

Every AI model has a context window — the maximum amount of text it can hold in working memory at one time. This includes your message, the conversation history, and any files or search results passed to the model. Once the context window is full, the model begins forgetting the earliest content. Understanding these limits helps you know when to start a fresh session or break a task into smaller parts.

ℹ
Tokens vs lines of code
Models measure context in tokens, not lines. Roughly: 1 line of code ≈ 10-20 tokens, 1 page of text ≈ 500 tokens, 1,000 tokens ≈ 750 words.
Tool Model Context Window ≈ Lines of Code ≈ Pages of Text Notes
Local AI / main.py qwen2.5:3b (router) 32K tokens ~1,600 ~64 Only classifies your input, not your files
Local AI — CODE qwen2.5-coder:7b 32K tokens* ~1,600 ~64 *Ollama default. Max possible is 128K
Local AI — MATH deepseek-r1:7b 32K tokens ~1,600 ~64 Best for self-contained problems
Local AI — GENERAL llama3.2:3b 128K tokens ~6,400 ~256 Largest context in your stack
Open WebUI Depends on selected model 32K–128K ~1,600–6,400 ~64–256 Same models, same limits. History counts toward context
Aider qwen2.5-coder:7b 32K tokens* ~1,600 ~64 Smarter about what it loads — only adds relevant files
Continue.dev (chat) qwen2.5-coder:7b 32K tokens* ~1,600 ~64 Includes open file + conversation history
Continue.dev (autocomplete) qwen2.5-coder:7b ~1,024 tokens ~50 ~2 Only looks at immediate surrounding code
TTS Kokoro (82M) No hard limit — ~Any length Processes text in chunks — just gets slower with long text
ComfyUI SDXL-Turbo 77 tokens (prompt) — ~half a page Image prompts should be concise — long prompts get truncated
File Converter (Pandoc) No AI — direct conversion Unlimited Any size Any size No context limit — converts entire files regardless of size

What happens when you hit the limit?

In chat (main.py / WebUI)
Oldest messages forgotten
The model silently drops the earliest messages in the conversation to fit new input. It won't warn you — answers may become inconsistent.
Start fresh conversation
In Aider
Files get truncated
If you add too many files, Aider warns you and may truncate some. Use /add selectively — only add files relevant to the current task.
Add fewer files
In ComfyUI
Prompt gets cut off
SDXL clips prompts at 77 tokens silently. Keep image prompts concise — quality keywords beat long sentences.
Keep prompts short

How to increase context in Ollama (advanced)

Ollama defaults to 32K tokens for most models. You can increase it by setting the context size when running a model, but it uses more VRAM:

terminal — increase context window 
# Run model with larger context (uses more VRAM)
OLLAMA_NUM_CTX=65536 ollama run qwen2.5-coder:7b

# Or set it permanently in ~/.ollama/config
# Note: going above 32K may cause OOM on your 8GB card
⚠
VRAM increases with larger contexts
Doubling the context window roughly doubles the VRAM needed for the KV cache. On an 8GB card, going above 32K tokens with a 7B model may cause out-of-memory errors. Stick to the default 32K unless you have a specific need and sufficient VRAM headroom.

17 — Troubleshooting
Common Problems
Solutions to issues you may encounter

Agents & Router

!
Router triggers FILE for casual file mentions
The router uses strict keywords — only "convert", "create a file", "make a file", "export", "save as" trigger FILE. Casual mentions like "my python file" route to CODE or GENERAL. If still happening, restart the agent session.
!
First response is very slow
Normal — the router model takes 2–4 seconds to load into VRAM on first run. Subsequent questions in the same session are faster.
!
Venv not activating / crewai not found
Run source ~/.venvs/agents/bin/activate.fish manually, then python ~/agents/main.py. Or use the agents alias which handles activation automatically.

PDF / File Conversion

!
pdflatex not found / lmodern.sty missing
Install the full texlive stack: sudo pacman -S texlive and press Enter to select all 22 packages. The agent uses xelatex as the PDF engine which is included in the bundle.
!
~/... path not found
The agent calls os.path.expanduser() automatically. But if using pandoc directly, always use the full absolute path: /home/username/Documents/file.txt instead of ~/Documents/file.txt.

TTS / Kokoro

!
TTS Error: numpy version conflict
Kokoro requires numpy>=2.0.2. If something downgraded it, fix it: activate agents venv then run pip install "numpy>=2.0.2". Verify with python -c "import kokoro_onnx; print('ok')".
!
No audio output from TTS
Check sounddevice: python -c "import sounddevice; print(sounddevice.query_devices())". Make sure your audio output is set correctly in system settings.

Aider

!
unrecognized arguments: --ollama-api-base
Remove the ollama-api-base line from ~/.aider.conf.yml. The file should only contain one line: model: ollama/qwen2.5-coder:7b. Newer aider versions don't use that flag.
!
Aider conflicts with crewai or kokoro
Never install aider into the agents venv. Aider lives at ~/.venvs/aider/. If numpy got downgraded in the agents venv, fix it: pip install "numpy>=2.0.2".
!
fastfetch shows on every aider ! command
Wrap fastfetch in an interactive check in ~/.config/fish/conf.d/30-autostart.fish: replace fastfetch with if status is-interactive / fastfetch / end.
!
GitHub push rejected — file too large
Model files exceed GitHub's 100MB limit. Add to .gitignore, remove from history with git filter-branch --force --index-filter "git rm --cached --ignore-unmatch kokoro-v0_19.onnx voices-v1.0.bin voices.bin" --prune-empty --tag-name-filter cat -- --all, then git push --force.

Ollama & Services

!
Out of VRAM / model fails to load
Don't run ComfyUI and agents simultaneously — ComfyUI uses ~6GB leaving nothing for Ollama. Close ComfyUI first. Check what's loaded: ollama ps.
!
Ollama not responding
Check status: systemctl status ollama. Restart: systemctl restart ollama. Verify models: ollama list.
!
SearXNG returns no results
Check Docker: docker ps | grep searxng. If not running: cd ~/searxng && docker-compose up -d. Then visit localhost:8888 to confirm.

The VRAM constraint
