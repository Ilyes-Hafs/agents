01 — Overview
System Architecture
How everything fits together
Router
qwen2.5:3b
Reads your input and decides which specialist to route to. Fast and lightweight — only 1.9GB.
Always loads first
Coding
qwen2.5-coder:7b
Handles all code: writing, debugging, explaining, optimizing. Used in Aider and Continue.dev too.
4.7GB VRAM
Math & Reasoning
deepseek-r1:7b
Distilled reasoning model. Step-by-step problem solving, logic, math. Punches above its weight.
4.7GB VRAM
General + Search
llama3.2:3b
Fast general assistant. Handles writing, translation, summaries, and search result synthesis.
2.0GB VRAM
ℹ
How routing works
The router model reads your message and outputs one word: CODE, MATH, SEARCH, TTS, FILE, or GENERAL. The right specialist model then loads and handles your request. Only one model is in VRAM at a time.

03 — AI Agents
main.py — The Brain
Your terminal-based multi-agent system

The main agent system is your primary interface for quick tasks. It automatically routes your question to the right specialist model without you needing to think about it.

How to use it

1
Activate the venv and start
The agents alias handles this automatically. Just type agents or run directly.
2
Type your question naturally
No need to specify which agent. Ask anything — "write a Python function", "what is 15% of 340", "search for latest Node.js version".
3
Watch the router decide
You'll see [Router → CODE] or similar, then the specialist responds. The loop continues — just keep asking.
4
Type exit to quit
Or Ctrl+C. The venv deactivates automatically.
terminal — example session 
# Start the system
python ~/agents/main.py

🤖 Local AI ready. Type your question (or 'exit'):

You: write a function to check if a number is prime
[Router → CODE]

You: search latest python version
[Router → SEARCH]
[Searching...]

You: say hello this is a test
[Router → TTS]
# Audio plays through speakers

You: exit
⚠
First response is slow
The router model needs to load into VRAM on first run (~2-4 seconds). Subsequent questions in the same session are faster.

04 — Open WebUI
Browser Chat Interface
ChatGPT-like UI running fully local

Open WebUI gives you a polished chat interface in your browser. Unlike main.py, here you manually pick which model to use from a dropdown. Best for longer conversations and uploading images.

How to use it

1
Open it
Type webui in terminal or go to http://localhost:8080. It auto-starts on boot.
2
Select a model
Click the model dropdown at the top. Choose based on your task: qwen2.5-coder:7b for code, llava:7b for images, llama3.2:3b for general chat.
3
Upload images with llava
Select llava:7b, then click the paperclip icon to attach an image. Ask anything about it: "describe this", "what's in the background", "read the text".
✓
Auto-starts on boot
Open WebUI runs as a systemd service. No need to start it manually — it's always available at localhost:8080.

07 — ComfyUI
Image Generation
Stable Diffusion XL Turbo on your 4060

ComfyUI is a node-based image generation interface. Your loaded model is SDXL-Turbo — optimized for speed, generates 1024×1024 images in a few seconds on your RTX 4060.

How to generate an image

1
Start ComfyUI
Type comfyui in terminal. It opens localhost:8188 in your browser automatically.
2
Load your workflow
Click Graph → Open file and select ~/agents/sdxl_workflow.json. This loads the pre-configured SDXL-Turbo workflow.
3
Edit the prompt
Find the CLIP Text Encode (Positive) node and type your description. Find the negative node to add what you don't want.
4
Click Run
The image generates and appears on the Save Image node. It's also saved to ~/ComfyUI/output/ as PNG.
ℹ
Best resolution for your GPU
Stick to 1024×1024 for best quality. Maximum safe resolution is around 1280×1280. Going higher may cause out-of-memory errors on your 8GB VRAM.
⚠
Manual start only
ComfyUI is not set to auto-start — it uses VRAM when running and would conflict with your AI models. Start it only when you need to generate images.

Prompt tips for SDXL-Turbo

  • Be descriptive: "a cyberpunk city at night, neon lights, rain, photorealistic, 8k"
  • Add style: "oil painting", "anime style", "watercolor", "cinematic lighting"
  • Negative prompt: always add "blurry, low quality, ugly, distorted, deformed"
  • SDXL-Turbo works best with 1-4 steps (it's a turbo model, not standard diffusion)

09 — Aider
Terminal Coding Assistant
AI that edits your actual files

Aider is the most powerful coding tool in your stack. Unlike Continue.dev (suggestions) or main.py (answers), Aider reads your entire codebase and directly edits files based on your instructions.

How to use it

1
Navigate to your project
cd ~/your-project-folder
2
Start aider
Type aider — it loads using qwen2.5-coder:7b automatically from your config.
3
Add files to context
Type /add filename.py to give aider access to specific files. Or /add *.py for all Python files.
4
Give instructions naturally
"Add error handling to the main function", "Refactor this to use async/await", "Fix the bug in line 42".
ℹ
Aider vs Continue.dev — when to use each
Use Continue.dev while actively writing code (autocomplete, quick questions). Use Aider for bigger tasks: refactoring, adding features, fixing complex bugs — things that touch multiple files.

12 — Limits
Context & Processing Limits
How much each tool can handle at once

Every AI model has a context window — the maximum amount of text it can read and reason about at once. Think of it as working memory. Once you exceed it, the model starts forgetting earlier parts of the conversation or file.

ℹ
Tokens vs lines of code
Models measure context in tokens, not lines. Roughly: 1 line of code ≈ 10-20 tokens, 1 page of text ≈ 500 tokens, 1,000 tokens ≈ 750 words.
Tool Model Context Window ≈ Lines of Code ≈ Pages of Text Notes
Local AI / main.py qwen2.5:3b (router) 32K tokens ~1,600 ~64 Only classifies your input, not your files
Local AI — CODE qwen2.5-coder:7b 32K tokens* ~1,600 ~64 *Ollama default. Max possible is 128K
Local AI — MATH deepseek-r1:7b 32K tokens ~1,600 ~64 Best for self-contained problems
Local AI — GENERAL llama3.2:3b 128K tokens ~6,400 ~256 Largest context in your stack
Open WebUI Depends on selected model 32K–128K ~1,600–6,400 ~64–256 Same models, same limits. History counts toward context
Aider qwen2.5-coder:7b 32K tokens* ~1,600 ~64 Smarter about what it loads — only adds relevant files
Continue.dev (chat) qwen2.5-coder:7b 32K tokens* ~1,600 ~64 Includes open file + conversation history
Continue.dev (autocomplete) qwen2.5-coder:7b ~1,024 tokens ~50 ~2 Only looks at immediate surrounding code
TTS Kokoro (82M) No hard limit — ~Any length Processes text in chunks — just gets slower with long text
ComfyUI SDXL-Turbo 77 tokens (prompt) — ~half a page Image prompts should be concise — long prompts get truncated
File Converter (Pandoc) No AI — direct conversion Unlimited Any size Any size No context limit — converts entire files regardless of size

What happens when you hit the limit?

In chat (main.py / WebUI)
Oldest messages forgotten
The model silently drops the earliest messages in the conversation to fit new input. It won't warn you — answers may become inconsistent.
Start fresh conversation
In Aider
Files get truncated
If you add too many files, Aider warns you and may truncate some. Use /add selectively — only add files relevant to the current task.
Add fewer files
In ComfyUI
Prompt gets cut off
SDXL clips prompts at 77 tokens silently. Keep image prompts concise — quality keywords beat long sentences.
Keep prompts short

How to increase context in Ollama (advanced)

Ollama defaults to 32K tokens for most models. You can increase it by setting the context size when running a model, but it uses more VRAM:

terminal — increase context window 
# Run model with larger context (uses more VRAM)
OLLAMA_NUM_CTX=65536 ollama run qwen2.5-coder:7b

# Or set it permanently in ~/.ollama/config
# Note: going above 32K may cause OOM on your 8GB card
⚠
VRAM warning for large contexts
Doubling the context window roughly doubles the VRAM needed for the KV cache. On your 8GB card, going above 32K tokens with a 7B model may cause out-of-memory errors. Stick to 32K unless you have a specific need.

How to change the default permanently

1
Open the TTS script
nano ~/agents/tts.py
2
Find the default voice line
Look for voice = "am_fenrir" in the parse_args function.
3
Change the value
Replace am_fenrir with any voice ID from the table above. Example: voice = "bm_george"
4
Also change the language default if needed
For British voices, change lang = "en-us" to lang = "en-gb" on the line below.
5
Save and test
Ctrl+X → Y → Enter to save. Then tts hello to test.
~/agents/tts.py — change these two lines 
def parse_args(args):
    # ... other code ...
    voice = "am_fenrir"   # ← change this to any voice ID
    lang  = "en-us"      # ← change to "en-gb" for British voices
    speed = 1.2          # ← adjust speed (1.0 = normal)

14 — Customization
Swapping AI Models
How to change which model handles which task

Swap a model in main.py

1
Pull the new model first
ollama pull newmodel:size — wait for download to finish.
2
Open main.py
nano ~/agents/main.py
3
Find the models section at the top
Look for the # ── Models ── comment block with the LLM definitions.
4
Change the model name
Replace the model string. Example: change "ollama/qwen2.5-coder:7b" to "ollama/codellama:7b"
~/agents/main.py — models section 
# ── Models ──────────────────────────────────────────────
router_llm  = LLM(model="ollama/qwen2.5:3b",         ...)  # router
coder_llm   = LLM(model="ollama/qwen2.5-coder:7b",   ...)  # coding
math_llm    = LLM(model="ollama/deepseek-r1:7b",     ...)  # math
general_llm = LLM(model="ollama/llama3.2:3b",        ...)  # general

Swap Aider's model

~/.aider.conf.yml 
model: ollama/qwen2.5-coder:7b   # ← change this
ollama-api-base: http://localhost:11434

Swap Continue.dev models

~/.continue/config.json 
{
  "models": [{
    "model": "qwen2.5-coder:7b"  // ← change this
  }],
  "tabAutocompleteModel": {
    "model": "qwen2.5-coder:7b"  // ← and this
  }
}
